# PD Bot Game • Cookie run — MuMuPlayer ADB Automation

โปรแกรมหน้าตา GUI (พร้อมแปลงเป็น .exe ได้) สำหรับควบคุม ADB ผ่าน MuMuPlayer
เพื่อทำงานอัตโนมัติในเกม CookieRun: สแกนหาจอ MuMuPlayer ที่เปิดอยู่ → เชื่อมต่อ ADB →
เปิดเกม → ล็อกอินโหมด Dev → สุ่มหาสัตว์เลี้ยงที่กำหนด → บันทึกผล/เก็บบัญชี → วนต่อไอดีถัดไป
จนครบเป้าหมาย พร้อม Log แสดงการทำงานแบบเรียลไทม์

## โครงสร้างโปรเจกต์

```
CookieRun/
  config/config.yaml         # ค่าคอนฟิกทั้งหมด (path, threshold, referral, run, license, ฯลฯ)
  templates/                 # รูปภาพอ้างอิงสำหรับตรวจสอบหน้าจอแต่ละขั้นตอน
    login/                   #   - หน้าจอล็อกอิน/โหมด Dev
    pet_reroll/               #   - หน้าจอสุ่มสัตว์เลี้ยง/เพชรหมด/ลบบัญชี/ออกจากระบบ
    treasure_reroll/          #   - หน้าจอสุ่มสมบัติ (รอรายละเอียดเพิ่ม)
  src/
    core/                     # ADB, สแกนหา MuMuPlayer, template matching, config, logging
    automation/                # Step/StepRunner, game_flow, pet_reroll, treasure_reroll,
                                # worker_control (pause/resume/cancel), stats (ตัวนับ)
    data/recorder.py            # บันทึกผลลง CSV/txt
    gui/
      app.py                    # หน้าตาโปรแกรมหลัก (BotApp) - ดูภาพตัวอย่างที่ผู้ใช้ส่งมา
      automation_worker.py       # รันงานอัตโนมัติแยกเธรดต่อจอ ประสานกับ GUI
      log_queue_handler.py        # ส่ง log จากเธรดพื้นหลังเข้าคิวให้ GUI แสดงผล
  tools/
    list_instances.py          # สแกน/แสดงรายการ MuMuPlayer ที่เปิดอยู่ (บรรทัดคำสั่ง)
    calibrate.py                 # เครื่องมือตัดรูปหน้าจอจริง -> เซฟเป็น template
  run.py                        # เวอร์ชันบรรทัดคำสั่ง (ไม่มีหน้าตา GUI)
  gui_main.py                   # จุดเริ่มโปรแกรม "หน้าตา" (GUI) <-- ใช้ตัวนี้
  build_exe.bat                 # สร้างไฟล์ .exe ด้วย PyInstaller
  requirements.txt
```

## การติดตั้ง

1. ติดตั้ง Python 3.11 ขึ้นไป (ต้องรองรับ `X | Y` type hints)
2. ติดตั้งไลบรารีที่ต้องใช้:

   ```
   pip install -r requirements.txt
   ```

3. เปิด MuMuPlayer (เปิดอย่างน้อย 1 อินสแตนซ์) ให้ระบบ Android บูตเสร็จก่อน

## วิธีรันโปรแกรม (หน้าตา GUI)

```
python gui_main.py
```

จะเปิดหน้าต่างโปรแกรม "PD Bot Game • Cookie run" ตามภาพตัวอย่างที่คุณส่งมา
ทุกปุ่ม/ช่องกรอกเชื่อมกับระบบจริงแล้ว (ไม่ใช่ mockup) ยกเว้นจุดที่ยังรอรายละเอียดเกมจากคุณ
(ดูหัวข้อ "สิ่งที่ต้องเติมเพิ่ม" ด้านล่าง)

## วิธีสร้างไฟล์ .exe

รันไฟล์ `build_exe.bat` (ดับเบิลคลิก หรือรันผ่าน cmd) จากโฟลเดอร์ `CookieRun/`

- จะได้โปรแกรมที่ `dist\PD_Bot_Game_CookieRun\PD_Bot_Game_CookieRun.exe`
- สคริปต์จะคัดลอกโฟลเดอร์ `config/` และ `templates/` ไปไว้ข้าง .exe ให้อัตโนมัติ
  (สร้าง `data_output/` และ `logs/` ให้ด้วย)
- **สำคัญ:** ต้องคัดลอกทั้งโฟลเดอร์ `dist\PD_Bot_Game_CookieRun\` ไปด้วยกันเสมอ
  (ไม่ใช่แค่ไฟล์ .exe อย่างเดียว) เพราะโปรแกรมต้องใช้ไฟล์ `config/config.yaml` และรูป
  `templates/` ที่อยู่ข้าง ๆ ตัว .exe ในการทำงาน และจะบันทึกผลงาน/Log ไว้ในโฟลเดอร์เดียวกันนี้ด้วย
- ถ้าต้องการไอคอนโปรแกรม ให้วางไฟล์ `assets/icon.ico` ไว้ก่อนรัน build_exe.bat
  (ไม่มีก็รันได้ปกติ แค่ใช้ไอคอนเริ่มต้นของ Windows)

## วิธีใช้งานหน้าโปรแกรมแบบละเอียด

หน้าโปรแกรมแบ่งเป็น 4 ส่วนหลัก ตรงกับภาพตัวอย่างที่คุณส่งมา:

### 1. แถบหัวโปรแกรม (บนสุด)

| องค์ประกอบ | คำอธิบาย |
|---|---|
| 🍪 PD Bot Game • Cookie run | ชื่อโปรแกรม/เวอร์ชัน |
| 🤍 ปั้มทำใจ-ชวนเพื่อน | สวิตช์ลัด ผูกกับตัวเลือก "ส่งคำเชิญเพื่อน (referrer) ก่อนเปิด" ในช่องตั้งค่า (ค่าเดียวกัน กดจากตรงไหนก็ได้) |
| 🎁 รีไอดี-สุ่มขอ | สวิตช์ลัด ผูกกับตัวเลือก "สมัครไอดีใหม่..." ในช่องตั้งค่า |
| 👑 VIP - ลงทะเบียนแล้ว 1/1 | **แสดงผลอย่างเดียว** อ่านจาก `config.yaml` (`license.registered_devices` / `max_devices`) ยังไม่มีระบบตรวจสอบสิทธิ์การใช้งานจริงอยู่ด้านหลัง (ดูหัวข้อ "สิ่งที่ต้องเติม" ถ้าต้องการระบบนี้จริง) |

### 2. ช่องตั้งค่า (ซ้ายบน) "⚙ ตั้งค่า"

- **Referral link** — ลิงก์ชวนเพื่อน/referral ที่จะเปิดก่อนเข้าเกม (ถ้าเปิดใช้ตัวเลือกด้านล่าง)
- **เป้าหมายรวม (ไอดี)** — จำนวนไอดีรวมที่ต้องการทำ **รวมทุกจอที่เลือกไว้พร้อมกัน** เมื่อทำครบจะหยุดเองทุกจอ
- **วิธีเก็บเมื่อจบรอบ** — เลือกว่าเมื่อไอดีหนึ่ง "เพชรหมด" (ไม่เจอสัตว์เลี้ยงที่ต้องการ) จะ:
  - 🚪 **ออกจากระบบ** — กดออกจากระบบในเกมแล้วเริ่มไอดีใหม่ (ต้อง config ปุ่ม logout ในเกมก่อน)
  - 🗑 **ลบข้อมูลเกม** — ลบบัญชี/ล้างข้อมูลแอปทั้งหมด (ปลอดภัยกว่า เพราะใช้ `pm clear` ของ ADB ได้เสมอแม้หาปุ่มในเกมไม่เจอ)
- **ส่งคำเชิญเพื่อน (referrer) ก่อนเปิด** — ถ้าติ๊ก จะเปิดลิงก์ Referral link ผ่านหน้าเว็บ/ดีพลิงก์ก่อนเปิดเกมทุกรอบไอดีใหม่
- **สมัครไอดีใหม่ (สุ่มจบ/เก็บ - เก็บ new_accounts.txt)** — ถ้าติ๊ก จะบันทึกไอดีใหม่ทุกรอบลงไฟล์
  `data_output/new_accounts.txt` (ปัจจุบันบันทึกเป็นรหัสอ้างอิงตามเวลา เนื่องจากยังไม่มีวิธีอ่าน UID จริงจากในเกม)

กด **เริ่ม** ครั้งแรก โปรแกรมจะบันทึกค่าที่กรอกไว้ทั้งหมดกลับลงไฟล์ `config/config.yaml`
ให้อัตโนมัติ (เปิดโปรแกรมใหม่ครั้งหน้าจะจำค่าที่ตั้งไว้)

### 3. ช่องจอ MuMu instances (ซ้ายล่าง) "🖥 จอ (MuMu instances)"

- **🔍 สแกนจอ** — สแกนหา MuMuPlayer ที่เปิดอยู่ทั้งหมดในเครื่อง (ใช้เวลาประมาณ 10-20 วินาที)
  ผลลัพธ์จะแสดงในรายการด้านล่าง พร้อมพอร์ต ADB และชื่อหน้าต่างที่จับคู่ได้
- **✅ เลือกทั้งหมด** — เลือก (หรือยกเลิกเลือกทั้งหมดถ้าเลือกอยู่แล้ว) ทุกจอในรายการ
  ถ้าไม่เลือกอะไรเลยตอนกด "เริ่ม" โปรแกรมจะใช้ **ทุกจอที่สแกนพบ** โดยอัตโนมัติ
- **🗑 ลบข้อมูลเกม** — ลบข้อมูลเกม (pm clear) ของจอที่ **เลือกไว้ในรายการ** ทันที
  (ใช้แยกจากการรันงานอัตโนมัติ เช่น อยากเคลียร์จอทิ้งก่อนเริ่มใหม่)
- ถ้าไม่พบจอเลย จะมีข้อความเตือน "⚠ ไม่พบจอ — เปิด MuMu ก่อนแล้วสแกน"

### 4. สถานะงาน + Log (ขวา) "💾 สถานะงาน" / "📋 Log การทำงาน"

- ตัวเลข 4 ตัว: **เริ่ม** (จำนวนไอดีที่เริ่มไปแล้ว) / **สำเร็จ** (เจอสัตว์เลี้ยงที่ต้องการ) /
  **ไม่สำเร็จ** (เพชรหมด/error) / **เหลือ** (เหลืออีกกี่ไอดีถึงจะครบเป้าหมาย) — อัปเดตสดทุก 0.5 วินาที
  รวมทุกจอที่กำลังทำงาน
- แถบความคืบหน้า (progress bar) คำนวณจาก เริ่ม / เป้าหมายรวม
- **Log การทำงาน** — แสดงบันทึกการทำงานแบบเรียลไทม์ พร้อม timestamp ทุกบรรทัด
  แต่ละบรรทัดจะมีชื่อจอกำกับไว้ (เช่น `[Cookie - 01] เปิดเกม...`) เพื่อแยกว่าข้อความมาจากจอไหน
  - dropdown "รวมทุกจอ" — เลือกดู log เฉพาะจอใดจอหนึ่ง หรือ "รวมทุกจอ" เพื่อดูทั้งหมด
  - 🗑 ล้าง — ล้างข้อความ log ทั้งหมดในหน้าจอ (ไม่กระทบไฟล์ log จริงที่ `logs/automation.log`)

### 5. แถบควบคุมด้านล่าง

| ปุ่ม | การทำงาน |
|---|---|
| ▶ เริ่ม | เริ่มทำงานกับจอที่เลือก (หรือทุกจอถ้าไม่เลือก) ตามเป้าหมายที่ตั้งไว้ |
| ⏸ หยุด | หยุดชั่วคราว (จะหยุดที่ขั้นตอนปัจจุบัน ไม่ตัดตอนกลางการแตะหน้าจอ) |
| ⏵ ทำต่อ | ทำงานต่อจากที่หยุดไว้ |
| ⏹ ยกเลิก | ยกเลิกการทำงานทั้งหมด ทุกจอจะหยุดอย่างปลอดภัยแล้วกลับสู่สถานะ "พร้อม" |

สถานะปัจจุบันแสดงด้านซ้ายของแถบนี้ (● พร้อม / ● กำลังทำงาน... / ● หยุดชั่วคราว / ● กำลังยกเลิก...)

## ⚠️ สิ่งที่คุณต้องเติมเพิ่มก่อนใช้งานได้จริง

โครงสร้างโปรแกรมทั้งหมด (สแกน ADB, ควบคุมอินพุต, ตรวจจับรูปภาพ, retry/error-handling,
หน้าตา GUI, บันทึกผล, สร้าง .exe) ทำงานได้ครบและผ่านการทดสอบกับ MuMuPlayer จริงแล้ว
**แต่ยังไม่มีข้อมูลเฉพาะของเกม CookieRun** เนื่องจากต้องรอรายละเอียดหน้าจอ/ขั้นตอนจากคุณ
รายการที่ต้องเติม:

1. **`config/config.yaml`**
   - `game.package_name` — ยืนยันชื่อแพ็กเกจ CookieRun เวอร์ชันที่คุณใช้จริง (ตอนนี้ตั้งเป็น
     `com.devsisters.cookierunkingdom` ซึ่งเป็นค่าเดา)
   - `game.target_pet` — ชื่อ/คีย์สัตว์เลี้ยงที่ต้องการ (ต้องตรงกับชื่อไฟล์ template ที่จะเพิ่มในข้อ 3)
   - `referral.link` — ลิงก์ referral จริงของคุณ

2. **ขั้นตอนในเกม (แก้โค้ดตรงนี้เมื่อได้รายละเอียด)**
   - `src/automation/game_flow.py :: build_login_steps()` — ปุ่ม/หน้าจอตอนล็อกอินโหมด Dev จริง
   - `src/automation/pet_reroll.py` — ปุ่มสุ่ม (`build_roll_step`), เงื่อนไข "เพชรหมด"
     (`gems_exhausted_templates`), ปุ่มลบบัญชี (`build_delete_account_steps`),
     ปุ่มออกจากระบบ (`build_logout_steps`)
   - `src/automation/treasure_reroll.py` — ยังเป็น placeholder ทั้งหมด รอรายละเอียดขั้นตอน "สุ่มสมบัติ"

3. **รูปภาพ template** — เพิ่มไฟล์ `.png` ในโฟลเดอร์ `templates/<category>/` ตามชื่อที่โค้ดอ้างอิง
   ใช้เครื่องมือช่วยตัดรูป:

   ```
   python tools/calibrate.py --instance 0 --category login --name dev_mode_button
   ```

   ดูรายชื่อไฟล์ที่ต้องมีทั้งหมดได้ใน `templates/README.md`

4. **ระบบ VIP/ลงทะเบียนเครื่อง** (ถ้าต้องการใช้งานจริง) — ปัจจุบันเป็นแค่ตัวเลขที่ตั้งไว้ใน
   `config.yaml` (`license.*`) ไม่มีระบบตรวจสอบ/ล็อกจำนวนเครื่องจริงอยู่ด้านหลัง
   ถ้าต้องการระบบนี้จริง (เช่น จำกัดจำนวนเครื่องที่ใช้ได้ต่อไลเซนส์ ผ่านเซิร์ฟเวอร์ตรวจสอบ)
   แจ้งรายละเอียดเพิ่มเติมได้ จะช่วยออกแบบให้

5. **วิธีอ่าน "ไอดี/UID" ของบัญชีจริง** — ตอนนี้ระบบใช้ timestamp เป็นรหัสอ้างอิงชั่วคราว
   (ใน `PetRerollController._current_account_id()` และ `new_accounts.txt`) เพราะยังไม่รู้ว่า
   จะอ่าน UID จริงจากหน้าจอเกมได้จากที่ไหน (ต้องใช้ OCR หรือมีข้อความ UID ที่ template จับได้)

## หมายเหตุเกี่ยวกับ MuMuPlayer

- โปรแกรมจะพยายามหา `MuMuManager.exe` ก่อน เพื่อดึงพอร์ต ADB และหน้าต่างของทุก instance
  โดยตรง (แม่นยำที่สุด รองรับ multi-instance) - ทดสอบแล้วใช้ได้กับ MuMuPlayer ที่ติดตั้งจริง
- ถ้าหา `MuMuManager.exe` ไม่เจอ หรือเวอร์ชันที่ใช้ไม่คืนพอร์ต ADB มาให้ (บางเวอร์ชันเป็นแบบนี้)
  จะ fallback ไปไล่สแกนพอร์ตตามกฎมาตรฐานของ MuMu (instance 0 = พอร์ต 16384, instance ถัดไป +32)
  ซึ่งทดสอบแล้วใช้งานได้จริงเช่นกัน
- ถ้าทั้งสองวิธีหาไม่เจอ ให้กำหนดพาธ `mumu_manager_path` หรือ `adb_path` เองใน
  `config/config.yaml`
